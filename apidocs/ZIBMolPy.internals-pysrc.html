  3  """ 
  4  Classes for working with internal coordinates. 
  5   
  6  The int File Format 
  7  =================== 
  8          An internals file (int-file) contains the definition of the internal coordinates used in the simulation. 
  9          So far, dihedral angles (dihedrals) and distances (linears) are supported. 
 10   
 11          The first line of an int-file contains the list of relevant dihedrals defined by the four atoms involved, e. g.:: 
 12                  (0-1-2-4),(1-2-3-4), ... 
 13   
 14          The second line of an int-file contains the list of relevant linears defined by the two atoms involved, e. g.:: 
 15                  (0-4),(20-46), ... 
 16                   
 17  Atom Indices 
 18  ============ 
 19  Please note: 
 20   
 21          1. Atom indices in int-files start with zero. 
 22           
 23          2. Atom indices in int-files refer to the absolut atom indices as can be found in the coordinate file of the complete system, and not on a per-molecule topology basis. 
 24           
 25   
 26  Labels 
 27  ====== 
 28          Each internal coordinate comes with a standard label stating its type and index. Optional custom labels can be specified within the int-file as:: 
 29                  (0-1-2-4){'label':'cat'},(1-2-3-4), ... 
 30   
 31          which will overwrite the standard label "dih_000" with "cat". 
 32   
 33  Weights and Offsets 
 34  =================== 
 35          Angle coordinates can take on only values between -2S{pi} and +2S{pi}. 
 36          But distance coordinates can in principle take on any positiv value. 
 37          This can lead to problems where distance coordinates are overly dominate. 
 38           
 39          Therefore, distance coordinates also offer a weight and an offset paramter:: 
 40                  (0-4){'weight':2.0, 'offset':1.5},(20-46){'label':'puppy', 'weight':2.0, 'offset':-0.4} 
 41           
 42          See also L{LinearCoordinate}. 
 43   
 44   
 45  Custom Coordinate 
 46  ================= 
 47  One can create his own subclasses of L{InternalCoordinate}. 
 48   
 49  They can be addressed in the int-file by providing a 'type' argument e.g.:: 
 50          (3-1-4){'type':'Funny', 'foobar':'cat'},... 
 51   
 52  This will call the following constructor:: 
 53          FunnyCoordinate(3, 1, 4, foobar='cat') 
 54           
 55  Besides writting a subclass one also has make some adjustments to the code  
 56  in L{zgf_create_nodes}, which determines the associated gromacs-restraints. 
 57
